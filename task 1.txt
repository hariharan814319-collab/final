==========================================
CURRENT BACKEND STATUS
==========================================

The backend no longer stores uploaded files inside:

uploads/profilePhotos/
uploads/aadhaar/
uploads/licenses/
uploads/degrees/

Instead, every uploaded file is uploaded directly to Cloudinary using multer-storage-cloudinary.

MongoDB now stores complete Cloudinary URLs such as:

https://res.cloudinary.com/<cloud_name>/image/upload/v1234567890/phoenix/profilePhotos/profile.jpg

instead of

uploads/profilePhotos/profile.jpg

Therefore the frontend must NEVER build local upload URLs anymore.

NEW LOGIC
====================================================

Cloudinary already returns a complete URL.

Example:

doctor.profilePhoto

contains

https://res.cloudinary.com/......

Therefore:

OLD

img.src =
API_URL + "/" + doctor.profilePhoto

NEW

img.src =
doctor.profilePhoto

----------------------------------------------------

OLD

documentLink.href =
API + "/" + doctor.degreeDocument

NEW

documentLink.href =
doctor.degreeDocument

----------------------------------------------------

OLD

licensePreview.src =
backendURL +
doctor.licenseDocument

NEW

licensePreview.src =
doctor.licenseDocument
